# templates — Reusable Templates

## Purpose

General-purpose, program-wide document templates — distinct from the
phase-specific templates already embedded in individual phase folders
(e.g.,
[`02-dependency-analysis/templates/`](../02-dependency-analysis/templates/README.md),
[`05-storage-migration/08-migration-execution-checklist.md`](../05-storage-migration/08-migration-execution-checklist.md)).
These live here because they're used across every phase, not tied to one.

## Owner

Migration Program Lead.

## Templates

| Template | Used When |
|---|---|
| [`change-request-template.md`](change-request-template.md) | Any change to scope, budget, or timeline requiring formal approval per the charter's amendment log |
| [`meeting-notes-template.md`](meeting-notes-template.md) | Any program meeting — stakeholder interviews, phase gate reviews, wave retrospectives |
| [`status-report-template.md`](status-report-template.md) | Weekly/monthly status reporting per [`00-project-overview/04-timeline-and-phases.md`](../00-project-overview/04-timeline-and-phases.md) |
| [`risk-register-entry-template.md`](risk-register-entry-template.md) | Adding a new risk to the register maintained in [`documentation/`](../documentation/README.md) |

## How to use

Copy the relevant template, fill it in, and store the completed document
in [`documentation/`](../documentation/README.md) or
[`logs/`](../logs/README.md) as appropriate — these files are blank,
reusable starting points, not filled examples.
